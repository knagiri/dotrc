## Bash ツールのコマンド制約

### 避けるべきシェル構文

以下のシェル構文を含むコマンドは、許可設定（allowlist）にマッチしないため **毎回ユーザーの手動承認が必要になる**。
自動実行の妨げになるので使用しないこと。

| 構文 | 名称 | 例 |
|------|------|-----|
| `$()` | コマンド置換 | `git commit -m "$(cat msg.txt)"` |
| `\|` | パイプ | `git log \| head -5` |
| `&&` | AND チェーン | `ruff check && pyright` |

### 代替手段

- **複数コマンドの実行**: 個別の Bash ツール呼び出しに分けるか、`;` や改行で連結する。依存関係がなければ並列呼び出しで効率化できる。
- **コマンド置換 `$()`**: シェル変数に事前格納する、または一時ファイルを経由する。
- **パイプ `|`**: 専用ツール（Grep, Read 等）や、コマンド自体のオプション（`git log -5` 等）で代替する。
